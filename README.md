
# API Client TUI

Интерактивный Python-клиент для OpenAPI сервисов.

Скрипт загружает OpenAPI схему, показывает список доступных методов и позволяет вызывать HTTP API прямо из TUI без ручного набора команд в shell.

## Возможности

- загрузка OpenAPI по адресу сервиса, адресу `/docs` или прямому адресу `/openapi.json`
- навигация по методам и полям через клавиатуру
- отправка запросов из интерфейса
- ввод path, query, header, cookie параметров
- ввод request body
- настройка авторизации прямо в TUI
- перезагрузка схемы без перезапуска программы

## Запуск

Из корня workspace:

```bash
python -m api_client http://localhost:8000
```

Из каталога api_client эта же команда тоже поддерживается:

```bash
cd api_client
python -m api_client http://localhost:8000
```

Можно передавать любой из вариантов:

```bash
python -m api_client http://localhost:8000
python -m api_client http://localhost:8000/docs
python -m api_client http://localhost:8000/openapi.json
```

Если схема не содержит `servers`, запросы будут отправляться на тот адрес, который был передан при запуске.
Если `servers` в OpenAPI присутствует, клиент использует первый сервер из схемы как базовый URL для запросов.

## Управление

- Стрелки вверх и вниз: перемещение по спискам
- Enter: открыть метод, изменить поле, подтвердить ввод, отправить запрос
- PgUp/PgDn: прокрутка длинного ответа на экране запроса
- a: открыть настройки авторизации
- u: изменить URL сервиса или OpenAPI
- r: заново загрузить OpenAPI схему и обновить экран
- q: выйти из программы
- Esc: отменить ввод в поле

## Как работать

1. Запусти клиент с адресом сервиса на localhost.
2. На первом экране выбери нужный endpoint стрелками.
3. Нажми Enter, чтобы открыть форму запроса.
4. Заполни параметры и body.
5. Выбери пункт Send request и нажми Enter.
6. Ответ сервера появится в нижней части экрана.

## Авторизация

Встроены следующие режимы авторизации:

- none
- basic
- bearer
- header
- query

Что это значит:

- basic: логин и пароль, клиент сам сформирует заголовок Authorization
- bearer: токен будет отправлен как Bearer token в Authorization
- header: можно указать произвольный заголовок и значение
- query: можно указать имя и значение query-параметра, например api_key

Настройка авторизации:

1. Нажми a на экране методов или формы запроса.
2. На поле Mode нажимай Enter, чтобы переключать режим.
3. Заполни нужные поля.
4. Выбери Save and go back.

## Request Body

Если метод принимает `application/json`, тело можно ввести в JSON виде.

Пример:

```json
{
  "name": "demo",
  "enabled": true
}
```

Если введён некорректный JSON, клиент покажет ошибку и не отправит запрос.

## Обновление схемы

Клавиша r:

- повторно скачивает OpenAPI схему
- обновляет список методов
- пытается сохранить текущую выбранную операцию, если она ещё существует

Клавиша u:

- позволяет поменять URL сервиса без выхода из программы
- после изменения сразу выполняет новую загрузку схемы

## Зависимости

Клиент использует только стандартную библиотеку Python:

- curses
- urllib
- json
- argparse

Отдельные pip-пакеты не требуются.

## Ограничения

- интерфейс рассчитан на терминал с поддержкой curses
- длинные ответы прокручиваются клавишами PgUp/PgDn; на очень маленьком экране может страдать читаемость
- используется базовый разбор OpenAPI схемы без генерации Python-классов клиента

## Файлы

- `__main__.py` — запуск через `python -m api_client`
